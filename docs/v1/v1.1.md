# V1.1 — Cierre profesional

## Objetivo

Cerrar una versión estable para Railway, mejorar documentación y metadatos, y eliminar dependencias accidentales de base de datos.

## Cambios principales

- README específico del portfolio con instalación y deploy.
- PHP mínimo alineado en `^8.4`.
- Variables de entorno sin base de datos:
  - `SESSION_DRIVER=file`
  - `CACHE_STORE=file`
  - `QUEUE_CONNECTION=sync`
- Documentación de `APP_URL` y `ASSET_URL` para Railway.
- Corrección del escaneo de Tailwind mediante:

  ```css
  @source '../**/*.blade.php';
  ```

- Canonical y `og:url` generados desde `config('app.url')`.
- JSON-LD corregido para emitir una clave `@context` válida.
- Menú móvil con etiqueta dinámica, cierre con `Escape` y devolución de foco.
- Tests deterministas para CV presente y ausente.
- Eliminación de la migración del script Composer `setup`.

## Deploy

URL de producción:

```text
https://portfolio-nicolas-santa-ana-production.up.railway.app
```

`ASSET_URL` se configuró con la URL pública para que Railway genere correctamente los enlaces de CSS y JavaScript compilados por Vite.

## Validación de cierre

- 6 tests y 27 aserciones correctas.
- Pint correcto.
- Build de Vite correcto.
- Canonical, Open Graph, JSON-LD, CSS y CV verificados en producción.
- Sin secretos ni artefactos generados rastreados por Git.

## Resultado

V1.1 quedó cerrada como base estable sobre la que se planifica la V2 por slices independientes.
